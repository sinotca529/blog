エディション

  • Java SE
    • 最低限の API
  • Java EE (Enterprise Edition)
    • SE + 大規模システムに必要なライブラリ
  • Java ME (Micro Edition)
    • 組み込み・モバイル用途
  • JRE : JVM + 標準クラスライブラリ
  • JDK : JRE + 開発ツール (コンパイラ、デバッガなど)
  • 標準クラスライブラリ : 基本ライブラリ + 統合ライブラリ + UIライブラリ

Java EE (ないしエディション) の必要性がよくわからない。 C/C++, Rust などはプロジェクトごとに必要なライブラリを取ってくるわけで、 わざわざエディションを分ける動機がいまいちわからない。 Rust でいうno_std に近いのだろうか。

用語

クラスパス
.java ファイル同様、 .class ファイルもパッケージ階層に沿ったパスに配置します。 この起点をクラスパスといいます。
モジュールパス
モジュールの起点となるパスです。
jar ファイル
.class のあるディレクトリを固めたものです。

javac コマンド

アプリのコンパイル

javac \
  --source-path <依存先 java ファイルの探索先> \
  -d <生成した class ファイルの配置先> \
  -cp <依存先 jar/class ファイルの探索先>
  <コンパイル対象>

--source-path で指定した場所にある .java ファイルのうち、コンパイル対象が依存するものだけがコンパイルされます。 そのため、アプリを作る場合は --source-path にコードの場所を指定し、コンパイル対象に main 関数を書いた .java ファイルを指定すればよいのです。

ライブラリのコンパイル

コンパイル対象に $(find src -name "*.java") を指定すれば全件コンパイルできるでしょうが、 通常は何かしらのビルド管理ツールを使用するでしょう。

モジュールのコンパイル (Java 9)

コンパイル対象に module-info.java が含まれている場合、モジュールが作成されます。

javac \
  -d <生成したモジュールの配置先> \
  <module-info.java の場所> \
  <コンパイル対象>

java コマンド

ソースファイルモード (Java11)

ソースコードをその場でコンパイルして実行するモードです。 中間コードはメモリ上に保持され、実行後に .class ファイルは残りません。

java Main.java 1 2 3

これは次のようなイメージです。

javac -d <memory> Main.java
java -cp <memory> Main 1 2 3
