Spring Framework

アノテーション

@SpringBootApplication (複合)
@EnableAutoConfiguration, @ComponentScan, @SpringBootConfiguration が有効になります。
@EnableAutoConfiguration
Spring Boot の自動構成メカニズムを有効にします。 例えば HSQLDB が jar の依存関係にあるならそれを自動で Bean 化するなどします。 DB であれば、自分で DataSource Bean を作ることで自動構成を回避できます。
@ComponentScan
アプリケーションが配置されているパッケージで @Component のスキャンを有効にします。 スキャンは @ComponentScan がついたクラスのあるパッケージから再帰的に実行されます。
@SpringBootConfiguration
?
@Configuration
@Configuration 付きのクラスにある @Bean メソッドは、 DI コンテナに登録するインスタンスの作成に使われます。 また、 @Configuration がついたクラスのあるファイルを Bean 定義ファイルと呼びます。
@EnableBatchProcessing
Spring Batch 5.0 においては、自動構成の内容をカスタムするためにこのアノテーションを使います。 このアノテーションをつけると Runner などを自分で定義する必要が生じます。 @EnableBatchProcessing のかわりに DefaultBatchConfiguration を継承したクラスでも構成内容をカスタムできます。 @EnableBatchProcessing と DefaultBatchConfiguration の併用はできません。
