SQL

ISO/IEC 9075-2:2023 は有償である。 代わりに JIS X3005-1:2014 と JIS 3005-2:2015 を参照する。

ブール型は unknown 値を取る

ブール (boolean) データ型の値は，真 (true) 又は偽 (false) のいずれかとする。 不定 (unknown) の真理値がナル値によって表現されることもある。

ブール型における NULL は unknown を表すらしい。

NULL 値と他の値の比較は常に unknown

ナル値は，幾つかの文脈中で，他のどの値とも等しくもなく，等しくなくもない， すなわち，与えられたどの値とも等しいかどうかが不定 (unknown) であるが， 複数のナル値は，一緒にして扱う。 例えば，<GROUP BY句>は，全てのナル値を一緒にして扱う。

NULL と NULL の比較も不定である。

(NULL = NULL) IS NULLtrue

副問い合わせはトランザクションを開始しうる

現行SQLトランザクションがないならば， トランザクション開始文 (transaction-initiating statement) の実行又は <副問合せ>の評価がトランザクションを開始する。

トランザクションは時刻印を持つ

(4.36.3) SQLトランザクションは， 処理系定義の時刻印型の値であるトランザクション時刻印 (transaction timestamp) をもち， その値は，このSQLトランザクション中のSQLデータ変更文の実行によって修正される行があるならば， それらの行のシステム時刻期間開始列及びシステム時刻期間終了列の値を設定するために用いる。 トランザクション時刻印は，そのトランザクション中でいずれかのSQLデータ変更文が実行される前にSQL処理系によって設定され， 一度設定されると，そのSQLトランザクションの間，変更されないでそのままとする。

カーソル

データを同時に触るトランザクションを「並行SQLトランザクション」と呼ぶ

(4.36.3) 異なるSQLエージェントによって開始され，同じSQLデータ又はスキーマをアクセスし， かつ，時刻で重なり合うSQLトランザクションは，並行SQLトランザクション (concurrent SQL-transaction) とする。

単一行服問い合わせは0行の時にNULLを返す。

単一引数の集約関数は、何も残らなかった場合に NULL を返す。 ただし、 COUNT は 0 を返す。

COUNT(*) は全列 NULL の行もカウントに含める。 その他の COUNT(e) は、 e が NULL 以外に評価されるものの数を数える。 たとえば COUNT(1) は COUNT(*) に等しく、 COUNT(NULL) は 0 となる。

ORDER BY は文末に1回だけ書ける。 UNION などをしている場合、 ORDER BY で利用できる列名は初めの SELECT で指定したものだけである。

Oracleでは、ソート時に NULL が無限大として扱われる。標準SQLでは規定なし。

索引の利用はオプティマイザ次第であり、 索引を作成したからと言って利用されるとは限らない。

BFILE は OS 上に配置したバイナリファイル (画像等) へのロケータを格納する型。 読み取り専用。 SELECT の結果はロケータ。 なお、ロケータとは、ディレクトリオブジェクトとファイル名の組み合わせのこと。

DML

マルチテーブル INSERT (Oracle の機能)

複数の表に一度に挿入できる。 ただし、リモート表への挿入はできない。

無条件でインサートする。

-- table1, table2 の両方に値を入れる
INSERT ALL
    INTO table1 (col1) VALUES (val1)
    INTO table2 (col1) VALUES (val2)
SELECT ... FROM ...;

条件付きでインサートする。

-- 条件に合致するものすべてに INSERT する
INSERT ALL
    WHEN col1 > 100 THEN INTO table1 VALUES (...)
    WHEN col2 = 'A' THEN INTO table2 VALUES (...)
SELECT ... FROM ...;

INSERT ALL ではなく INSERT FIRST を書くと、 初めてヒットした行にだけ格納する。

MERGE

条件に一致する場合に UPDATE、一致しない場合に INSERT をする構文。

MERGE INTO target_table t
USING source_table s ON (t.id = s.id)
WHEN MATCHED THEN
    UPDATE SET t.col1 = s.col1
WHEN NOT MATCHED THEN
    INSERT (id, col1) VALUES (s.id, s.col1);

WHEN MATCHED と WHEN NOT MATCHED はどちらか一方だけでも問題ない。 また、 ON に指定した列は更新できない。

特殊な表

DUAL表

DUAL表はSYSユーザーのスキーマにあり、すべてのユーザーがDUALの名前でアクセスできる。 VARCHAR2(1) の DUMMY列 を持ち、1行だけデータを持つ。データの値は ‘X’ である。

ビュー

SELECT 文に名前を付けたもので、実体はない。 単純なビューであれば、ビューを介して DML を実行できる。 DISTINCT, GROUP BY, JOIN, ORDER BY, SUM などが入った複雑なビューに DML は適用できない。

ビューの作成時に WITH CHECK OPTION をつけると、DML 発行時に、変更後のデータがビューに含まれるかの検証が入る。 つまり、変更後の行がビューに現れない場合にエラーとなる (DELETE を除く)。

一時表

一時表は、セッションやトランザクション中だけデータを保持する表。 ビュー, シノニム, 索引も作成できる。 データはトランザクション終了時またはセッション終了時に削除されるが、テーブルは削除されない。 データを見れるのは、自セッションのみ。

-- トランザクション終了時にデータを削除
CREATE GLOBAL TEMPORARY TABLE tmp_table ( col NUMBER ) ON COMMIT DELETE ROWS;

-- セッション終了時にデータを削除
CREATE GLOBAL TEMPORARY TABLE tmp_table ( col NUMBER ) ON COMMIT PRESERVE ROWS;

プライベート一時表 (Oracle 18c～)

表の定義もセッションまたはトランザクションの終了時に削除される一時表。 普通の一時表がデータディクショナリで管理されるのに対し、プライベート一時表はメモリ上で管理される。

外部表

データベースの外部にあるファイルを扱う。 CREATE TABLE (...) ORGANIZATION EXTERNAL (...) で作成する。 外部ファイルを扱うため、ディレクトリオブジェクトへのアクセス権限が必要。

アクセスドライバを2つから選択する。 デフォルトでは ORACLE_LOADER が利用される。

  • ORACLE_DATAPUMP : バイナリのダンプファイルを利用。ロード・アンロードが可能
  • ORACLE_LOADER : CSV などのフラットファイルを利用。ロードのみ可能 (デフォルト)
