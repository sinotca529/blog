(Java) インタフェース

メソッドの宣言

メソッドには次の種類があります。

種類書き方説明
抽象メソッド[abstract] [public] void foo();実装を持たない。
デフォルトメソッドdefault [public] void foo() { ... }インスタンスメソッドとして動作。常に public。オーバーライド可能。
静的メソッドstatic [public] void foo() { ... }インタフェース名から呼び出す。常に public。オーバーライド不可。
プライベートメソッドprivate void foo() { ... }Java 9 以降で使用可。内部ユーティリティ用で、外部や実装クラスから呼べない。
プライベート静的メソッドprivate static void foo() { ... }Java 9 以降で使用可。static/default メソッドから呼び出せる内部ユーティリティ。

メンバの宣言

メンバは暗黙的に public static final になります。 修飾子を明示的に書くことはできますが、 public, static, final 以外はつけられません。

public interface I {
  // 暗黙的に public static final
  int A = 10;

  // 明示的に書いても良い
  static final int B = 10;

  // エラー
  private int C = 10;
}

備考

特に意味はありませんが、 abstract interface I {} は合法です。

なぜ default キーワードが必要なのか？

おそらくコンパイラの都合でしょう。 もし default キーワードがない場合、コンパイラは {} か ; までトークンを読み進めないと、それが抽象メソッドなのか具象メソッドなのかを判定できません。 default キーワードがあれば、先読みなしに抽象/具象を特定できます。

他にも、 public static final や public abstract を省略できるのは、定数・メソッドの修飾子は1種類しか許可されないので省略を許してもコンパイラが先読みしなくて済むからだと解釈できます。 また、抽象クラスでは抽象メソッドに abstract をつける必要があるのも同じように解釈できます。 default を必須にするか abstract を必須にするかでダブスタ感がありますが、そこは目をつぶりましょう。

実現 (implements)

実現すべきメソッドを自分で定義する必要はありません。 親クラスから継いだものでも構いません。

interface I { int method(); }
class B { public int method() { return 1; } }
class E extends B implements I {}

void main(String[] args) {
  E e = new E();
  System.out.println(e.method()); // 1
}
