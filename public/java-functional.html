(Java) 関数型インタフェース

関数型インタフェースは抽象メソッドを1つだけ持つインタフェースを指し、 @FunctionalInterface アノテーションをつけて明示します。 アノテーションは省略可能ですが、つけておくと関数型インタフェースの条件を満たしていない場合にコンパイルエラーとすることができます。 なお、 抽象メソッドとは別に private メソッドや static メソッドを持つことは許されています。

@FunctionalInterface
interface Adder<T> { T add(T a, T b); }

ラムダ式

ラムダ式を評価すると関数型インタフェースを実装した無名クラスのインスタンスが得られます。

// Adder を実装した無名クラスのインスタンスを生成
Adder<Integer> r = (a, b) -> a + b;
System.out.println(r.getClass()); // class Main$$Lambda/0x00007f828b154408
System.out.println(r.add(2, 3));  // 5

引数名として、スコープにある変数と同じ名前は使えません。

var a = 10;
Adder<Integer> r = (a, b) -> a + b;

ラムダ式は、式の外にある変数のうち、宣言に final がついている (または付けてもコンパイルが通る) ものを参照できます。 これはラムダ式固有の制約ではなく、ローカルクラスの制約に由来するものと考えられます。

var offset = 10;
Adder<Integer> r = (a, b) -> a + b + offset;
System.out.println(r.add(2, 3)); // 15

あらかじめ用意されている主要なインタフェース

主要な関数型インタフェースを次の表に示します。 Rust の Fn トレイト的なものがないため、色々使い分ける必要があります。

インタフェースメソッド
Supplier<T>T get()
Consumer<T>void accept(T)
BiConsumer<T, U>void accept(T, U)
Predicate<T>boolean test(T)
BiPredicate<T, U>boolean test(T, U)
Function<T, R>R apply(T)
BiFunction<T, U, R>R apply(T, U)
UnaryOperator<T>T apply(T)
BinaryOperator<T>T apply(T, T)
Runnablevoid run()
Callable<V>V call()
  • Runnable と Callble の覚え方
    • Run は走ったっきり帰ってこないので戻り値なし。 Call は呼びかけへの応答があるので戻り値ありと覚える。
