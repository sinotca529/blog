(Java) JDBC

JDBC (Java DataBase Connectivity) は RDBMS を統一的に扱うための仕組みです。 DB の操作に必要なインタフェースや共通実装を提供します。 個別 RDBMS 向けの実装は、ベンダーが提供するドライバ (ライブラリ) で補填します。

データベース接続

try {
    // 接続開始
    Connection con = DriverManager.getConnection("jdbc:derby://localhost:1527/data/Sample");
    // 接続終了
    con.close();
} catch (SQLException e) {
    throw new RuntimeException(e);
}

SQL の実行

検索をするには executeQuery を利用します。

var sql = "SELECT * FROM emps WHERE age > ?";
try (PreparedStatement ps = con.prepareStatement(sql)) {
    ps.setInt(1, 30); // パラメタ位置は1始まり

    ResultSet rs = ps.executeQuery();
    while (rs.next()) {
        System.out.println(rs.getInt(1) + ", " + rs.getString(2));
    }

} catch (SQLException e) {
    throw new RuntimeException(e);
}

挿入・更新・削除をするには executeUpdate を利用します。

var sql = "INSERT INTO emps VALUES (?, ?)";
try (PreparedStatement ps = con.prepareStatement(sql)) {
    ps.setString(1, "Taro"); // パラメタ位置は1始まり
    ps.set(1, 30); // パラメタ位置は1始まり

    int n = ps.executeUpdate(); // 処理件数
    System.out.println(n);
} catch (SQLException e) {
    throw new RuntimeException(e);
}

execute は検索/挿入・更新・削除のいずれも扱えます。 真偽値を返し、 true なら ps.getResultSet で executeQuery 相当の結果を、 false なら ps.getUpdateCount で executeUpdate 相当の結果を得られます。

また、 setX 系メソッドは executeX 系メソッドの実行後も発行できます。

var sql = "INSERT INTO emps VALUES (?, ?)";
try (PreparedStatement ps = con.prepareStatement(sql)) {
    ps.set(1, 30);

    ps.setString(1, "Taro");
    ps.executeUpdate();

    ps.setString(1, "Jiro"); // OK
    ps.executeUpdate();

} catch (SQLException e) {
    throw new RuntimeException(e);
}
