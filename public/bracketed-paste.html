Bracketed Paste を使ってターミナルに複数行のテキストを貼り付ける

貼り付け大暴走

改行の含まれた文字列をターミナルに貼り付けると、コマンドが実行されてしまうことがあります。 例えば次のような感じになったりします。

user@pc1:~
$ #include <stdio.h>                  // クリップボードの内容
zsh: parse error near `\n'            //   #include <stdio.h>
user@pc1:~                            //   int main(void) {
$ int main(void) {                    //       puts("Hello world!");
zsh: unknown file attribute: v        //       return 0;
user@pc1:~                            //   }
$     puts("Hello world!");
dquote>     return 0;
dquote> }

うっかり変なコマンドを実行しちゃいそうで怖いですね。

Bracketed Paste

これを防ぐ仕組みが bracketed paste です。 Bracketed Paste が有効になっていると、複数行の貼り付けをしてもコマンドの実行が起こりません。

user@pc1:~
$ #include <stdio.h>
int main(void) {
    puts("Hello World!");
    return 0;
}

Bracketed Paste のしくみ

仕組みは単純です。 貼り付け内容の前後に特殊な文字をつけることで、 単純なキー入力とクリップボードからの貼り付けを区別しています。

  • ESC[200~ : 貼り付けの開始
  • ESC[201~ : 貼り付けの終了
